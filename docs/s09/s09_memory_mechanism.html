三个关键设计细节

设计点说明
两条加载路径 索引常驻 SYSTEM：MEMORY.md 只是文件名 + 描述的一行行链接，极轻量（≤200 行），可被 prompt cache 缓存，几乎零额外成本。
内容按需注入：只有 LLM 判定相关的记忆（最多 5 条）才读取完整内容注入当前 user turn，不破坏 prompt cache 结构。
压缩前快照 压缩管线会截断/改写消息，导致细节丢失。但提取记忆需要完整上下文。
解决：压缩前先 pre_compress = messages.copy()，压缩后 extract_memories(pre_compress) 用原始消息提取，保证完整性。
LLM 优先 + 关键词降级 select_relevant_memories 正常走 LLM side-query 做语义匹配；一旦网络异常或 JSON 解析失败，自动降级到关键词子串匹配，保证鲁棒性。
