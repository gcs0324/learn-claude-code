s18 Worktree Isolation 目录隔离

Git worktree · 任务-目录绑定 · 互不干扰

1. 问题：多队友共享同一目录

痛点: s17 中 Alice 和 Bob 都在同一个目录下工作。Alice write_file("config.py", ...)，Bob 也 write_file("config.py", ...)。两个人改同一个文件，互相覆盖。s15-s17 解决了"谁干什么"和"怎么通信"，但没解决"在哪干"。

2. Git Worktree 方案

无隔离 (s17)Worktree 隔离 (s18)
工作目录共享同一个目录各自独立目录 + 独立分支
文件冲突互相覆盖互不干扰
回滚分不清谁的改动按分支独立回滚
绑定无Task.worktree 字段绑定

6. 新增方法详解

s18 在 s17 基础上增加了 5 个方法 + 1 个目录 + 1 个 Task 字段，聚焦于目录隔离：

层次方法/类型作用
目录WORKTREES_DIR.worktrees/ 目录，每个 worktree 一个子目录
Task 扩展Task.worktree新增字段：绑定任务到 worktree 名称
安全层validate_worktree_name()拒绝 ../ 路径穿越、非法字符，只允许 [A-Za-z0-9._-]{1,64}
创建层create_worktree()validate → git worktree add -b wt/name → bind_task → log_event
绑定层bind_task_to_worktree()task.worktree = name + save_task（不改状态，保持 pending）
清理层remove_worktree()git worktree remove + git branch -D wt/name
保留层keep_worktree()保留目录和分支，不清理
审计层log_event()记录 create/remove/keep 操作到 events.jsonl

7. 整体流程变化：s17 → s18

s17 工作流程（之前）

flowchart LR A["Lead 创建任务"] --> B["队友 claim → 共享目录工作"] --> C["完成"]

s18 工作流程（现在）

flowchart TD A["Lead: create_task"] --> B["Lead: create_worktree name task_id"] B --> C["bind_task_to_worktree: task.worktree = name"] C --> D["队友: scan_unclaimed_tasks → claim"] D --> E["队友: cd .worktrees/name/ → 工作"] E --> F{"完成?"} F -->|"清理"| G["remove_worktree"] F -->|"保留"| H["keep_worktree"]

三个关键设计细节

设计点说明
安全校验validate_worktree_name 拒绝 ../ 路径穿越和非法字符，只允许 [A-Za-z0-9._-]{1,64}。防止恶意 name 访问 worktree 目录外的文件。
绑定不改状态bind_task_to_worktree 只写 task.worktree 字段，不改变 task.status。任务保持 pending，等队友自己 claim。绑定和认领是独立的两个步骤。
Git 原生隔离每个 worktree 有独立的分支（wt/name），队友的改动在各自分支上。完成后可独立 merge 或丢弃，不会互相污染。
