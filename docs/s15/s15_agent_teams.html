1. 子 Agent vs 队友

s06 子 Agents15 队友

核心区别: s06 的子 Agent 是临时工——叫来干一件事就走了。s15 的队友是能通信、能协作的长期同事。Lead 通过 MessageBus 收发消息，队友跑在自己的 daemon 线程里。

三个关键设计细节

设计点说明
文件收件箱 为什么用文件而不是内存队列？文件跨线程可观察（debug 时直接 cat .mailboxes/alice.jsonl），真实 CC 也是文件收件箱。
read_inbox 是消费式的——读完就删文件，避免重复处理。
队友简化工具集 队友只有 4 个工具（bash、read_file、write_file、send_message），没有任务管理、cron 调度等。
真实 CC 的队友也有 TaskCreate、TaskUpdate，任务系统是团队共享的。
10 轮限制 + daemon 线程 教学版限 10 轮防止无限循环；真实 CC 用 idle loop：跑完一轮发 idle_notification，等 inbox 消息，收到后继续。
threading.Thread(daemon=True) 保证主线程退出时队友线程自动清理。

维度s14s15
Agent 数量11 Lead + N 队友线程
通信无MessageBus + .mailboxes/*.jsonl

新增方法cron 调度相关spawn_teammate_thread + send/check 工具

核心痛点单个 Agent 上下文有限多 Agent 并行 + 通信协作
s14 → s15 的关系: s14 提供 cron 调度（定时任务），s15 在此基础上增加多 Agent 团队协作。两个系统独立叠加：cron 管定时触发，团队管并行分工。Lead 可以 cron 定时检查队友进度，也可以 spawn 队友后继续处理自己的任务。
