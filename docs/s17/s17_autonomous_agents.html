s17 Autonomous Agents 自治Agent

空闲轮询 · 自动认领 · WORK → IDLE → SHUTDOWN

1. 问题：s16 队友被动等任务

痛点: s16 的每个队友等 Lead 手动分配任务。如果任务看板上有 10 个未认领任务，Lead 得手动 assign 10 次。队友应该自己看任务看板，发现没人做的任务就认领，做完再找下一个。

2. 三阶段生命周期

阶段行为退出条件
WORKinbox → LLM → 工具循环（bash/read/write/send_message + claim_task/complete_task）stop_reason != tool_use
IDLE每 5s 轮询 inbox + 任务板，有活就干，没活继续等60s 超时 或 收到 shutdown_request
SHUTDOWN发 summary 给 Lead，退出线程—

6. 新增方法详解

s17 在 s16 基础上增加了 3 个方法 + 2 个常量，聚焦于队友自治：

层次方法/类型作用
常量IDLE_POLL_INTERVAL5 秒轮询间隔，控制 IDLE 阶段检查频率
常量IDLE_TIMEOUT60 秒空闲超时，超时后队友自动 SHUTDOWN
轮询层idle_poll()核心空闲循环：每 5s 检查 inbox（优先）→ 扫描任务板 → 有活回 WORK / 超时 SHUTDOWN
扫描层scan_unclaimed_tasks()list_tasks → 过滤 pending + owner==None + can_start → 返回可认领列表
认领层auto_claim_and_work()claim 第一个可认领任务 → 注入 prompt 到 messages → 回到 WORK 循环

三个关键设计细节

设计点说明
inbox 优先idle_poll 先检查收件箱，再扫描任务板。Lead 的消息（如 shutdown_request）优先级高于新任务。
自动认领无需 Lead队友 scan_unclaimed_tasks → claim_task → 自动注入 prompt → 回到 WORK。Lead 只需创建任务，队友自己认领。
超时保护60 秒超时防止队友无限空闲。真实 CC 没有固定超时，用 idle loop 持续轮询直到收到 shutdown_request。
