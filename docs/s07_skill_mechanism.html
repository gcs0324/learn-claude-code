5. 各阶段总结

阶段 谁操作 做什么 token 成本
安装 开发者 在 skills/<name>/SKILL.md 写文件 0
启动 _scan_skills() 扫描目录，解析 frontmatter，构建注册表 0
注入 build_system() 把 skill 名称+描述写入 SYSTEM prompt ~100 / skill
触发 大模型 判断任务需要 skill，调用 load_skill(name) 0 额外
加载 load_skill() 从注册表取完整 SKILL.md 返回 ~2000 / skill

6. s07 模拟 vs 真实 Claude Code

对比维度 s07（模拟版） 真实 Claude Code
skill 目录在哪 SYSTEM prompt（每次请求都带） system-reminder 消息（harness 注入）
大模型如何触发 调用 load_skill(name) 工具 调用 Skill 工具
完整内容如何获得 程序返回 tool_result harness 直接注入到上下文
API 回合数 2 回合（先 load，再执行） 1 回合（Skill 调用后内容即就位）
触发方式 大模型"推理"需要哪个 description 写明了触发规则，按规则匹配
💡 核心差异：s07 的 load_skill 是普通工具，tool_result 必须回到 messages 等下一轮大模型才能读到——所以多了一轮。 真实 Claude Code 的 Skill 是 harness 级特殊工具，harness 拦截调用后直接把 skill 内容注入，大模型同一轮就能用——省了一轮。
